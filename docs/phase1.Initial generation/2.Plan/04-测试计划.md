# Outline2Page 测试计划

## 1. 测试目标

测试计划用于确保 Outline2Page 的每个模块可以独立验证，并在集成后满足需求文档与详细设计文档的规则。

核心目标：

1. Markdown 解析稳定。
2. 页面计划、页码、命名、布局可预测。
3. Figma 模板扫描和节点复制不破坏原模板。
4. 属性注入正确，缺失属性不中断。
5. 导航 SHOW / HIGHLIGHT 正确。
6. TOC `NAV_group` 可扩展。
7. 重新生成只覆盖插件生成的 Section。
8. UI 能展示预览、警告和生成报告。

## 2. 测试类型

| 类型 | 覆盖范围 | 工具 |
| --- | --- | --- |
| 单元测试 | 纯逻辑模块 | Vitest 或等价测试框架 |
| Mock 测试 | Figma API 适配模块 | 自定义 Figma Mock |
| 集成测试 | 主线程流程 | Figma Mock + 手动验证 |
| 手动验收 | 真实 Figma 文件 | Figma 桌面端或网页端 |
| 回归测试 | 已确认样例 | 固定 Markdown 和模板组合 |

## 3. 单元测试范围

### 3.1 OutlineParser

测试用例：

```md
《Vision》= 零售 SI 设计系统

# TOC

## 品牌总览
### 品牌定位
##### 核心信息
##### 视觉关键词
### 门店策略
##### 空间体验

## 应用规范
### 字体
### 色彩
```

预期：

1. vision = `零售 SI 设计系统`。
2. hasToc = true。
3. chapterCount = 2。
4. titleCount = 4。
5. stepCount = 3。
6. 无严重错误。

异常用例：

```md
### 没有章节的标题
##### 没有标题的小节
#### 未定义层级
```

预期：

1. 返回警告。
2. 不抛出异常。
3. 可继续生成可识别部分。

### 3.2 OutlineAnalyzer

测试矩阵：

| 输入结构 | requiredPageKinds |
| --- | --- |
| Vision + TOC + CHAPTER | COVER, TOC, CHAPTER |
| Vision + TOC + CHAPTER + TITLE | COVER, TOC, CHAPTER, TITLE |
| Vision + TOC + CHAPTER + TITLE + STEP | COVER, TOC, CHAPTER, TITLE, STEP |

检查：

1. 章节数正确。
2. 标题数正确。
3. 小节数正确。
4. estimatedPageCount 正确。

### 3.3 PagePlanner

用例：

1. 完整模板映射。
2. 缺少 STEP 模板。
3. 缺少 TOC 模板。
4. 只有 CHAPTER 的大纲。

检查：

1. 页面顺序正确。
2. skippedPages 正确。
3. 每个 PagePlanItem 的上下文正确。

### 3.4 PaginationService

用例：

```text
COVER
TOC
CHAPTER 1
TITLE 1.1
STEP 1.1.1
STEP 1.1.2
TITLE 1.2
CHAPTER 2
TITLE 2.1
```

预期：

1. TOC pageNumber = 1。
2. CHAPTER 1 pageNumber = 2。
3. STEP 1.1.2 pageNumber = 5。
4. CHAPTER 1 range = `02-06`。
5. CHAPTER 2 range = `07-08`。

### 3.5 NamingService

检查：

1. COVER 名称包含 `00_COVER`。
2. TOC 名称包含 `01_TOC`。
3. CHAPTER 名称包含章节文本。
4. TITLE 名称包含标题文本。
5. STEP 名称包含小节文本。
6. 换行、多空格被清理。

### 3.6 LayoutEngine

输入：

1. 所有模板尺寸为 `1920x1080`。
2. 两个章节。
3. 第一章两个 TITLE，每个 TITLE 两个 STEP。

检查：

1. 同一章节在同一 y。
2. 不同章节 y 差包含最大高度 + 100。
3. 同一 TITLE 下 STEP 间距为 100。
4. 不同 TITLE 小队间距为 200。
5. 节点不重叠。

### 3.7 Navigation 排序逻辑

显式后缀：

```text
NAV_STEP_02
NAV_STEP_01
```

预期排序：

```text
NAV_STEP_01
NAV_STEP_02
```

无后缀：

```text
NAV_STEP
NAV_STEP
NAV_STEP
```

预期：

1. 按 y 从小到大。
2. y 接近时按 x 从小到大。
3. 自动赋予逻辑序号 01、02、03。

## 4. Figma Mock 测试范围

### 4.1 TemplateScanner

Mock 节点：

1. `PAGE_TEMP：COVER` Frame。
2. `PAGE_TEMP：TOC` Component。
3. `PAGE_TEMP：STEP` Instance。
4. `普通 Frame`。

检查：

1. 只返回 3 个模板。
2. kindGuess 分别为 COVER / TOC / STEP。
3. 普通 Frame 被忽略。
4. 能读取属性名和 Text Layer 名称。

### 4.2 GeneratedSectionManager

Mock 当前 Page：

1. 有一个带 `outline2page.generated=true` 的 Section。
2. 有一个普通 Section。
3. 有一个 `PAGE_TEMP：CHAPTER`。

检查：

1. 只删除插件生成 Section。
2. 普通 Section 保留。
3. 模板保留。
4. 新 Section 有 pluginData。

### 4.3 NodeFactory

Mock 模板：

1. Frame。
2. Component。
3. Instance。

检查：

1. Frame 调用 clone。
2. Component 调用 createInstance。
3. Instance 调用 clone。
4. 节点名称正确。
5. 节点坐标正确。
6. 节点加入 Section。

### 4.4 PropertyInjector

Mock 生成节点包含：

1. InstanceNode，暴露 `PAGE_TITLE_TEXT`。
2. InstanceNode，暴露 `PAGE_PAGE_TEXT`。
3. TextNode，名称为 `PAGE_TITLE_TEXT`。
4. TextNode，名称为 `PAGE_VERSION_TEXT`。

检查：

1. 组件属性写入。
2. Text Layer 写入。
3. `TYPE` 不写入。
4. 缺失属性进入警告。
5. 字体加载失败进入警告。

### 4.5 NavigationInjector

Mock 导航项：

1. `NAV_CHAPTER`
2. `NAV_TITLE`
3. `NAV_STEP`
4. 多个无后缀 STEP。

检查：

1. SHOW 写入布尔值。
2. HIGHLIGHT 写入 `on/off`。
3. 当前页面对应项高亮。
4. 多余项隐藏。

### 4.6 TocExpander

Mock TOC 节点包含：

1. 一个 `NAV_group`。
2. 多个目录条目需求。

检查：

1. 复制足够数量的 `NAV_group`。
2. 写入 `TOC_CHAPTER_TEXT`。
3. 写入 `TOC_TITLE_TEXT`。
4. 写入 `TOC_NUM_TEXT`。
5. 写入 `TOC_PAGE_RANGE_TEXT`。
6. 报告 expandedCount。

## 5. 手动验收准备

### 5.1 Figma 测试文件

当前 Page 中准备模板：

```text
PAGE_TEMP：COVER
PAGE_TEMP：TOC
PAGE_TEMP：CHAPTER
PAGE_TEMP：TITLE
PAGE_TEMP：STEP
```

模板要求：

1. 模板保留在当前 Page。
2. TOC 模板内至少有一个 `NAV_group`。
3. 导航栏内有 `NAV_CHAPTER`、`NAV_TITLE`、`NAV_STEP`。
4. 至少一个组件实例暴露 `SHOW`。
5. 至少一个组件实例暴露 `HIGHLIGHT`。
6. 至少一个 Text Layer 名称等于 `PAGE_TITLE_TEXT`。

### 5.2 手动输入样例

```md
《Vision》= 品牌体验升级

# TOC

## 第一章 品牌基础
### 品牌定位
##### 用户洞察
##### 价值主张
### 视觉语言
##### 色彩系统
##### 字体系统

## 第二章 空间应用
### 门店动线
##### 入口体验
##### 陈列节奏
### 导视系统
```

### 5.3 预期结果

1. 创建一个 `Outline2Page_GENERATED` Section。
2. Section 被选中。
3. 模板节点不被移动、不被改名、不被删除。
4. 生成 COVER、TOC、CHAPTER、TITLE、STEP 页面。
5. `PAGE_VERSION_TEXT` 写入 `品牌体验升级`。
6. `PAGE_PAGE_TEXT` 写入实际页码。
7. `TOC_NUM_TEXT` 写入章节号。
8. `TOC_PAGE_RANGE_TEXT` 写入章节跨度。
9. 当前导航项 `HIGHLIGHT=on`。
10. 多余导航项 `SHOW=false`。

## 6. 覆盖生成测试

步骤：

1. 第一次生成。
2. 在生成 Section 外部放一个普通矩形。
3. 在生成 Section 内部手动改一个文字。
4. 第二次生成。

预期：

1. 旧生成 Section 被删除。
2. 新生成 Section 创建。
3. Section 外部普通矩形保留。
4. Section 内部手动修改被覆盖。
5. 报告显示覆盖旧 Section 数量。

## 7. 缺失模板测试

步骤：

1. 删除或不选择 `PAGE_TEMP：STEP`。
2. 输入包含 STEP 的大纲。
3. 点击生成。

预期：

1. COVER / TOC / CHAPTER / TITLE 正常生成。
2. STEP 页面被跳过。
3. 报告提示缺失 STEP 模板。
4. 插件不崩溃。

## 8. 缺失属性测试

步骤：

1. 使用没有 `PAGE_STEP_TEXT` 的 STEP 模板。
2. 输入包含 STEP 的大纲。
3. 点击生成。

预期：

1. STEP 页面正常生成。
2. 其他属性正常注入。
3. 报告提示缺少 `PAGE_STEP_TEXT`。
4. 插件不崩溃。

## 9. TOC 扩展测试

步骤：

1. TOC 模板只保留一个 `NAV_group`。
2. 输入多个 CHAPTER 和 TITLE。
3. 点击生成。

预期：

1. 插件复制 `NAV_group`。
2. TOC 行数足够显示目录。
3. 报告提示扩展数量。
4. 每行目录数据正确。

## 10. 导航无后缀测试

步骤：

1. 模板中的导航项命名为多个重复的 `NAV_STEP`，不加 `_01`。
2. 输入多个 STEP。
3. 点击生成。

预期：

1. 插件按视觉顺序赋序号。
2. 第一个 STEP 写入第一个导航项。
3. 第二个 STEP 写入第二个导航项。
4. 当前 STEP 高亮正确。
5. 报告可提示使用了自动排序。

## 11. 回归测试清单

每次关键修改后至少验证：

1. 完整大纲生成。
2. 只有 CHAPTER 的大纲生成。
3. 缺少 STEP 模板。
4. 缺少属性。
5. TOC 扩展。
6. 重新生成覆盖。
7. 模板不被修改。
8. Section 被选中。

## 12. 通过标准

测试通过需要满足：

1. 所有纯逻辑单元测试通过。
2. Figma Mock 测试覆盖主要节点操作。
3. 手动验收样例可完整生成。
4. 缺模板、缺属性、层级异常均不会导致插件崩溃。
5. 生成报告能准确说明创建、覆盖、跳过、警告和 TOC 扩展情况。
