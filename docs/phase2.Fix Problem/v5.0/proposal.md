# v5.0 目录页标题扩展与缺失属性误报修复方案

## 背景

用户在 Figma 中生成目录页后，目录卡片标题未完整写入，插件面板出现大量 `TOC_TITLE_SLOT_CLONE_FAILED`、`PROPERTY_TARGET_MISSING`、`MISSING_PROPERTY` 警告。

## 当前问题

- `TOC_TITLE_TEXT` 标题槽位不足时，当前逻辑直接 clone 标题槽位实例；真实模板中该槽位位于嵌套组件实例内，clone 可能失败，导致目录标题扩展停止。
- 普通属性注入器会跳过导航容器，但仍把导航专属属性计入缺失目标，产生 `PAGE_CHAPTER_TEXT (HUGE)` 等误报。

## 目标方案

- TOC 标题槽位不足时，优先扩展父级标题行或所属容器，避免直接 clone 嵌套内部实例失败。
- 普通属性注入缺失检查只统计普通内容写入目标，不把导航专属目标误报为缺失。
- 保持现有模板扫描、页面命名、字段映射和生成流程不变。

## 不做范围

- 不重新设计目录模板结构。
- 不改变用户字段映射 UI。
- 不引入持久化配置。

## 待确认问题

无。截图和现有代码已能确定该问题影响生成结果，应按 Phase 2 Fix 处理。

## 验收标准

- 当 `TOC_TITLE_TEXT` 位于嵌套 `NAV_item` 中且标题槽位不足时，目录标题可以扩展并写入。
- 普通属性注入不会因为跳过导航容器而报告导航属性缺失。
- 既有 TOC、导航、属性注入测试保持通过。
- `npm test`、`npm run typecheck`、`npm run lint`、`npm run build` 全部通过。

## 已知风险

- 真实 Figma 组件实例内部 clone 能力受组件结构约束，测试需覆盖父级容器 clone 的回退路径。
- 不能吞掉真正的内容属性缺失警告，只能过滤导航专属误报。
